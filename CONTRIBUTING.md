# Contributing

Thanks for taking a look at this profile repository.

## What belongs here

- Small fixes to `README.md` or `assets/` that keep the profile accurate and fast-loading.
- Workflow tweaks that respect GitHub Actions limits on your account.

## What to avoid

- Do not commit secrets (API keys, `.env`, private tokens).
- Prefer **release assets** for large/binary images referenced from the profile README (see the `readme-assets` release).

## Suggested flow

1. Open an issue describing the change (optional for tiny typos).
2. Fork (if you are not a collaborator) and branch from `main`.
3. Open a pull request with a short summary and screenshots if visuals change.

If you are the repo owner iterating solo, direct commits on `main` are fine; pull requests still help with review history.
